Remaining gaps

现在仍然缺什么

第一类：Tushare 用户最容易感知的缺口

  • 财务指标、财务审计意见、主营业务构成、财报披露日期。
  • 周线、月线、实时行情、更多分钟频率规范化、每日指标。
  • 基金持仓、基金经理、基金公司、基金规模、基金分红。
  • 港美股交易日历、复权行情、实时行情、财务指标。

第二类：Agent 投研解释力缺口

  • 政策库、互动问答、长新闻、新闻联播、更多公告分类。
  • 交易所异常波动、严重异常波动、重点提示证券。
  • 机构调研、券商盈利预测、行业/概念异动来源的多源融合。

第三类：大类资产覆盖缺口

  • 期权、债券回购、国债收益率曲线、柜台债报价。
  • 外汇基础和外汇行情。
  • 期货仓单、结算参数、主力连续合约等更完整期货链路。

Diagnosis

真正问题

问题一：MCP Schema 还不够机器可读

ENDPOINTS 有 path、method、params、max_page_size，但参数类型和必填主要在 docstring，返回字段 schema 不在注册表里。MCP 自动生成还需要更结构化的 catalog。

问题二：Tushare 用户迁移心智不足

Tushare 用户熟悉 daily、income、cashflow 这类 API 名。FTShare 命名更解释型，需要提供 Tushare 对照表和常见任务示例。

问题三：缺口不该平均补

不要为了追 258 个数量去补低优先级资产。第一阶段应该优先补股票投研、基金 ETF 深度和 Agent 解释材料。

Positioning

第一阶段定位建议

FTShare MCP v0.1 应该基于 176 个 SDK 接口做 API 级 MCP，而不是先做复杂投研工作流。

更准确的目标是：让 Tushare / AKShare 用户能理解，让 Claude Code / Cursor / Codex 能直接调用，让股票投研场景有足够完整的数据基础。
